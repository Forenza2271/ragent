## 一、项目概述

### 1.1 Ragent 是什么？

Ragent 是一个**企业级 RAG 智能体平台**，基于 Java 17 + Spring Boot 3 + React 18 + Milvus 2.6 构建。

它覆盖了 RAG 系统从文档入库到智能问答的全链路工程实现：
- **多路检索引擎**：意图定向检索 + 全局向量检索并行，结果经去重、重排序等后处理
- **意图识别体系**：基于树形结构的多级意图分类（领域→类目→话题）
- **问题重写与拆分**：多轮对话中自动补全上下文，复杂问题拆分为子问题
- **会话记忆管理**：滑动窗口 + 超阈值自动摘要压缩
- **模型路由与容错**：多模型候选、首包探测、自动降级切换
- **MCP 工具集成**：知识检索与业务系统工具调用无缝融合
- **文档入库流水线**：基于节点编排的 Pipeline（获取→解析→增强→分块→向量化）
- **全链路追踪**：每个环节都有 Trace 记录

### 1.2 项目技术栈

| 层面       | 技术选型                                    |
| ---------- | ------------------------------------------- |
| 后端框架   | Java 17、Spring Boot 3.5.x、MyBatis Plus    |
| 前端框架   | React 18、Vite、TypeScript                  |
| 关系数据库 | MySQL（20+ 张业务表）                       |
| 向量数据库 | Milvus 2.6                                  |
| 缓存/限流  | Redis + Redisson                            |
| 消息队列   | RocketMQ 5.x                                |
| 文档解析   | Apache Tika 3.2                             |
| 模型供应商 | 百炼、SiliconFlow、Ollama（本地）           |

## 二、核心概念解析

### 2.1 什么是 RAG？

**RAG（Retrieval-Augmented Generation）** = 检索增强生成

传统 LLM 的问题：
- 知识截止日期限制（训练数据有时效性）
- 幻觉问题（胡编乱造）
- 无法访问私有数据

RAG 的解决思路：
```
用户问题 → 检索相关文档 → 将文档作为上下文 → LLM 基于上下文生成答案
```

**类比理解**：LLM 是一个聪明但"闭卷考试"的学生，RAG 让他可以"开卷"——带着参考资料答题。

### 2.2 什么是 Agent？

**Agent（智能体）** = LLM + 工具调用 + 规划能力

Agent 能够：
1. 理解用户意图
2. 决定调用什么工具（搜索、计算、API调用等）
3. 根据工具返回结果继续推理
4. 最终给出答案

**Ragent 中的 Agent 体现**：
- 意图识别决定走"知识库检索"还是"MCP工具调用"
- MCP（Model Control Protocol）让 LLM 能调用业务系统

### 2.3 RAG 完整链路（Ragent 实现）

```
用户提问
    ↓
┌───────────────────────────────────────────────────────────┐
│ 1. 问题预处理                                               │
│    ├── 问题重写：补全多轮对话上下文                          │
│    └── 问题拆分：复杂问题分解为子问题                        │
└───────────────────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────────────┐
│ 2. 意图识别                                                 │
│    ├── 树形意图分类（DOMAIN → CATEGORY → TOPIC）            │
│    ├── 判断走 KB（知识库）还是 MCP（工具调用）               │
│    └── 歧义引导（置信度不足时主动询问用户）                   │
└───────────────────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────────────┐
│ 3. 多通道检索                                               │
│    ├── 意图定向检索：根据意图节点关联的知识库检索            │
│    ├── 向量全局检索：意图置信度低时触发全局搜索              │
│    ├── MCP工具调用：意图为MCP类型时提取参数并执行            │
│    └── 后处理流水线：去重 → Rerank重排序 → TopK截断         │
└───────────────────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────────────┐
│ 4. Prompt组装 & LLM生成                                     │
│    ├── 系统提示词 + 检索上下文 + 会话历史 + 用户问题         │
│    ├── 流式输出（SSE）                                       │
│    └── 模型路由：多候选 → 首包探测 → 失败自动降级            │
└───────────────────────────────────────────────────────────┘
    ↓
┌───────────────────────────────────────────────────────────┐
│ 5. 会话记忆                                                 │
│    ├── 保留最近N轮对话                                       │
│    └── 超阈值自动摘要压缩                                    │
└───────────────────────────────────────────────────────────┘
```

- **Spring Boot**：项目结构、依赖注入、AOP 都是标准用法
